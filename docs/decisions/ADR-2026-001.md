# ADR-2026-001: Tech stack keuze: Astro + FastAPI + Neo4j

> **Datum:** 2026-03-04
> **Status:** geaccepteerd

## Context

BeeHaive is een knowledge framework voor AI-solutions met twee gezichten: een content-zware publieke website (blogs, framework info, kennisbank) en een interactief trainingsportaal (OAuth, voortgang, betalingen). Daarnaast is er een knowledge graph nodig voor het opslaan en doorzoekbaar maken van knowledge items (YouTube transcripts, whitepapers, etc.) met relaties naar de 7 bouwstenen en 7 EU guardrails.

Constraints:
- EU data residency verplicht (AVG)
- Robin heeft Astro-ervaring
- Knowledge items moeten met complexe relaties doorzoekbaar zijn
- Platform moet groeien van MVP naar volwassen training platform

## Beslissing

- **Frontend:** Astro + React islands (Astro voor content, React voor interactieve delen)
- **Backend:** Python FastAPI (API + AI pipeline)
- **Knowledge Graph:** Neo4j Community Edition (self-hosted)
- **Relationele DB:** PostgreSQL (users, cursussen, betalingen)
- **Auth:** Auth.js via Astro
- **Betalingen:** Mollie (NL)
- **Hosting:** Hetzner (EU)
- **AI/LLM:** Claude API voor content generatie

## Rationale

- Astro is content-first en Robin kent het al; React islands voor interactieve delen
- FastAPI is het beste Python framework voor AI/ML integratie (async, typing, OpenAPI)
- Neo4j is de industriestandaard voor knowledge graphs met uitstekende Python tooling
- PostgreSQL voor gestructureerde data (users, cursussen) naast de graph
- Mollie is Nederlands en volledig EU-gebaseerd
- Hetzner garandeert EU data residency

## Alternatieven overwogen

### Alternatief 1: Next.js voor alles
- **Voordelen:** Een codebase, Auth.js native, groot ecosystem
- **Nadelen:** Overkill voor content pagina's, Robin moet het leren
- **Waarom niet gekozen:** Astro is beter voor content-first sites en Robin kent het al

### Alternatief 2: PostgreSQL + Apache AGE (in plaats van Neo4j)
- **Voordelen:** Een database minder, simpeler ops
- **Nadelen:** Veel beperkter in graph queries, minder tooling
- **Waarom niet gekozen:** Knowledge graph is kernfunctionaliteit, Neo4j is de standaard

### Alternatief 3: Stripe (in plaats van Mollie)
- **Voordelen:** Groter ecosystem, meer integraties
- **Nadelen:** Niet-NL, complexere EU data residency configuratie
- **Waarom niet gekozen:** Mollie is Nederlands, iDEAL native, volledig EU

## Consequenties

### Positief
- Sterke scheiding van concerns (content vs. API vs. data)
- Elke laag kan apart schalen
- Best-of-breed per domein

### Negatief
- Meerdere technologieen om te onderhouden
- Twee databases (Neo4j + PostgreSQL)
- Deploy complexiteit (meerdere services)

### Risico's
- Auth.js integratie in Astro is minder mature dan in Next.js
- Neo4j Community heeft geen enterprise features (clustering, etc.)
