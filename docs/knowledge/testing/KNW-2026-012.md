# KNW-2026-012: Test-isolatie via prefixed dataset in gedeelde database

> **Categorie:** testing
> **Datum:** 2026-03-11
> **Sessie-context:** Integration tests faalden na uitbreiding van de Neo4j seed data van 7 naar 34 items
> **Relevantie:** hoog

## Inzicht

Wanneer integration tests tegen een gedeelde database draaien, gebruik een geïsoleerd test-dataset met een unieke prefix (bijv. `_TEST_`). De fixture maakt de data aan vóór de tests en ruimt het op na afloop. Dit maakt tests onafhankelijk van de productie-data die in dezelfde database zit.

## Context

De BeeHaive integration tests draaiden tegen dezelfde Neo4j database als de productie seed. Tests hadden hardcoded counts (`assert len(result) == 1`) die braken zodra de seed uitgebreid werd van 7 naar 34 KnowledgeItems. De fout lag niet in de code maar in de testaanpak: tests waren gekoppeld aan de omvang van de productiedata.

## Geleerd

### Wat werkte
- `_TEST_` prefix op alle node-namen maakt cleanup eenvoudig (`WHERE n.name STARTS WITH '_TEST_'`)
- Klein known dataset (2 BuildingBlocks, 2 Guardrails, 3 KnowledgeItems) is voldoende voor alle query-tests
- Module-scoped fixture (setup + teardown) houdt de overhead laag
- Exacte count assertions zijn weer mogelijk op de bekende dataset

### Wat niet werkte
- Hardcoded counts op productiedata — breekt bij elke uitbreiding
- `>=` checks als alternatief — test je eigenlijk niets precies meer

### Waarom
Tests moeten deterministisch zijn. Een gedeelde, groeiende database is per definitie niet deterministisch qua omvang. Door een eigen dataset te beheren heb je controle over de exacte state.

## Toepassing

Pas dit patroon toe wanneer:
- Integration tests draaien tegen een gedeelde database (Neo4j, PostgreSQL, etc.)
- De database ook productie-seed data bevat die kan groeien
- Tests exacte resultaten moeten valideren (counts, specifieke namen)

Concrete stappen:
1. Definieer een klein test-dataset in `conftest.py` met unieke prefix
2. Maak een setup fixture die de data aanmaakt (module-scoped)
3. Maak een teardown die alles met de prefix verwijdert
4. Laat tests alleen refereren naar de prefixed data
