# KNW-2026-004: Pytest marker registratie bij afwijkende rootdir

> **Categorie:** testing
> **Datum:** 2026-03-04
> **Sessie-context:** Integration tests opzetten voor Neo4j query-laag in een monorepo structuur
> **Relevantie:** midden

## Inzicht

Wanneer `pyproject.toml` niet in de project root staat maar in een subdirectory (bijv. `backend/pyproject.toml`), vindt pytest de marker-registratie niet automatisch. Dit leidt tot `PytestUnknownMarkWarning` ondanks correcte configuratie in pyproject.toml.

## Context

BeeHaive heeft een monorepo structuur met `backend/pyproject.toml`. De `@pytest.mark.integration` marker was correct geregistreerd in `[tool.pytest.ini_options]`, maar pytest gaf warnings omdat het de root directory als rootdir gebruikte en de pyproject.toml niet vond.

## Geleerd

### Wat werkte
- Expliciet `-c backend/pyproject.toml` meegeven aan het pytest commando
- Combinatie met `--rootdir=backend` voor correcte pad-resolutie
- Volledige Makefile-target: `.venv/bin/pytest -c backend/pyproject.toml --rootdir=backend -m integration -v`

### Wat niet werkte
- Alleen marker registratie in `backend/pyproject.toml` zonder `-c` flag
- Verwachten dat pytest automatisch de juiste pyproject.toml vindt in een monorepo

### Waarom
Pytest zoekt pyproject.toml in de rootdir en parent directories. In een monorepo waar de config in een subdirectory staat, matcht de rootdir-detectie niet met de config-locatie. De `-c` flag overschrijft dit gedrag expliciet.

## Toepassing

Bij monorepo's met subdirectory-specifieke pytest configuratie, gebruik altijd:
```makefile
test:
	.venv/bin/pytest -c subdir/pyproject.toml --rootdir=subdir -v
```

Alternatief: plaats een minimale `pyproject.toml` of `pytest.ini` in de project root die verwijst naar de juiste testpaths.
