# KNW-2026-010: Iteratieve code review-rondes vinden steeds diepere issues

> **Categorie:** development
> **Datum:** 2026-03-09
> **Sessie-context:** 3 review-rondes op ingestion pipeline code
> **Relevantie:** midden

## Inzicht

Meerdere review-rondes op dezelfde code vinden telkens andere categorieën issues. Ronde 1 vindt structurele bugs (data-integriteit, ontbrekende constraints). Ronde 2 vindt threading, schema en conventie-issues. Ronde 3 vindt edge cases, AVG-compliance en performance. Plan minimaal 2 rondes voor kritieke code.

## Context

De ingestion pipeline doorliep 3 AI review-rondes met 10 agents per ronde. Elke ronde leverde 7 gefilterde findings op (van ~18 ruwe findings). Alle 21 findings waren uniek — geen herhalingen tussen rondes.

## Geleerd

### Wat werkte
- Fix-apply-review cyclus: pas alle fixes toe, draai dan opnieuw
- Agents vonden in latere rondes subtielere issues doordat de "laaghangend fruit" al weg was
- Review-rondes kosten ~5 minuten maar voorkomen bugs die dagen debugging kosten

### Wat niet werkte
- Verwachting dat één review-ronde voldoende is — zelfs na r1 bleven 14 nieuwe issues liggen

### Waarom
Review-agents hebben beperkte aandacht per ronde. Wanneer grove fouten aanwezig zijn, domineren die de output. Na het fixen daarvan worden subtielere patronen zichtbaar (edge cases, compliance, performance).

## Toepassing

- **Kritieke paden** (security, privacy, data-integriteit): minimaal 2 review-rondes
- **Standaard features**: 1 ronde meestal voldoende
- Stop wanneer een ronde geen HIGH/CRITICAL findings meer oplevert
- Houd bij: totaal findings per ronde, als de trend stijgt is er een structureel probleem
