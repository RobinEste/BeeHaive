# KNW-2026-022: Opbouwende timeline als storytelling pattern

> **Categorie:** development
> **Datum:** 2026-03-26
> **Sessie-context:** Herbouw van de workflow visualisatie van een statische React Flow graph naar een opbouwende links-naar-rechts timeline
> **Relevantie:** hoog
> **Reviewed:** 2026-03-27

## Inzicht

Een auto-play timeline die links-naar-rechts opbouwt is effectiever voor storytelling dan een statische graph die alles tegelijk toont. Blokjes verschijnen één voor één, scrollen mee, en de bezoeker kan klikken om terug te lezen. Dit patroon werkt voor elke product-demo of procesuitleg die een verhaal vertelt.

## Context

De eerste versie was een React Flow graph met alle nodes zichtbaar en handmatige "Volgende →" navigatie. Robin stelde voor: een flow die automatisch opbouwt, met alleen Start/Stop, horizontale scroll, en beschrijvingen onder elk blokje. Later werd dit verfijnd met 3 tabs (Waarom/Hoe/Wat) en per-stap Evaluation Loop.

## Geleerd

### Wat werkte
- Auto-play met pauzeer/hervat — de flow loopt vanzelf, bezoeker kan ingrijpen
- Horizontale scroll met `scrollIntoView` op het actieve blokje
- Focus-state los van visible-state: klikken op een eerder blokje toont de beschrijving zonder latere blokjes te verbergen
- Tab-navigatie voor meerdere perspectieven op dezelfde use case

### Wat niet werkte
- React Flow (187KB) voor een lineaire opbouwende flow — overkill, custom React (10.7KB) volstaat
- `scrollToEnd` bij elke stap — duwt eerdere blokjes uit beeld
- Handmatige "Volgende →" knoppen — te veel interactie voor een showcase

### Waarom
Een statische graph is een overzicht. Een opbouwende timeline is een verhaal. Voor sales/showcase context wil je een verhaal: begin → midden → eind, met de mogelijkheid om terug te gaan. De bezoeker voelt progressie.

## Toepassing

Gebruik dit patroon wanneer je een proces of traject wilt demonstreren aan een niet-technisch publiek. De combinatie van auto-play (passief kijken) + klikbare blokjes (actief verkennen) bedient beide gebruikerstypen. De tabs (Waarom/Hoe/Wat) zijn herbruikbaar voor elke use case die vanuit meerdere perspectieven verteld kan worden.
