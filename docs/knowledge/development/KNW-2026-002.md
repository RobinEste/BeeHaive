# KNW-2026-002: Neo4j tx.run() parameter naam 'query' is gereserveerd

> **Categorie:** development
> **Datum:** 2026-03-04
> **Sessie-context:** Query-laag bouwen voor BeeHaive knowledge graph (13 Cypher functies)
> **Relevantie:** hoog

## Inzicht

De Neo4j Python driver's `tx.run()` methode heeft `query` als eerste positioneel argument. Als je een Cypher parameter `$query` noemt en deze als keyword argument `query=value` meegeeft, krijg je een `TypeError: got multiple values for argument 'query'`.

## Context

De `search_items()` functie ontving een `query` parameter van de caller en gaf deze door aan `tx.run("... $query ...", query=query)`. Dit conflicteerde met de signatuur van `TransactionBase.run(query, **kwargs)`.

## Geleerd

### Wat werkte
- Hernoemen van de Cypher parameter naar `$term` en doorgeven als `term=query`
- Alternatief: een ander naam kiezen voor de Python parameter

### Wat niet werkte
- `tx.run("... $query ...", query=query)` geeft een TypeError

### Waarom
De Neo4j driver's `run()` signatuur is `run(query: str, **kwargs)` — `query` is een gereserveerde positional parameter. Elke Cypher parameter die via kwargs wordt meegegeven, mag niet `query` heten.

## Toepassing

Vermijd `$query` als Cypher parameternaam bij gebruik van de Neo4j Python driver. Gebruik alternatieven als `$term`, `$search_term`, of `$input`. Dit geldt voor alle functies die `tx.run()` of `session.run()` aanroepen.
