# KNW-2026-024: Library evaluatie: complexiteit vs. use case

> **Categorie:** development
> **Datum:** 2026-03-25
> **Sessie-context:** Keuze tussen React Flow en een custom React component voor de workflow visualisatie
> **Relevantie:** hoog
> **Reviewed:** 2026-03-27

## Inzicht

Een gespecialiseerde library die 95% van wat je nodig hebt out-of-the-box biedt, kan toch de verkeerde keuze zijn als je use case fundamenteel anders is dan waarvoor de library is ontworpen. React Flow is gebouwd voor interactieve node-grafieken; de workflow visualisatie is een opbouwende timeline. Het custom component was 18x kleiner (10.7KB vs 187KB) en beter geschikt.

## Context

React Flow (`@xyflow/react`) was geïnstalleerd voor de workflow visualisatie. De eerste versie gebruikte het als een statische graph met klikbare nodes. Toen de requirements veranderden naar een opbouwende timeline met auto-play, scroll en tabs, bleek React Flow meer in de weg te zitten dan te helpen. Een custom React component met CSS transitions deed alles wat nodig was.

## Geleerd

### Wat werkte
- Eerst bouwen met de library, dan evalueren of het past
- Library behouden voor toekomstig gebruik (framework graph, BB detail-pagina's) maar niet forceren voor deze use case
- Bundle size als concrete metric voor de trade-off

### Wat niet werkte
- Aannemen dat een graph-library geschikt is voor elke vorm van verbonden-blokjes visualisatie

### Waarom
Libraries zijn geoptimaliseerd voor hun primaire use case. React Flow excelleert in drag-and-drop node editors, zoom/pan, edge routing. Voor een lineaire, opbouwende timeline is dat overhead — zowel in bundle size als in API-complexiteit. De custom oplossing heeft minder code, minder bytes, en meer controle over het specifieke gedrag (auto-play, scroll, focus-state).

## Toepassing

Voordat je een library installeert: beschrijf je use case in één zin. Beschrijf de library's primaire use case in één zin. Als die twee zinnen niet op elkaar lijken, bouw het custom. Bewaar de library voor wanneer je wél de primaire use case nodig hebt.
