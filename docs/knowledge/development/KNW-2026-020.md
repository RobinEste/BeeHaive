# KNW-2026-020: Gemini model generatie-sprong: altijd nieuwste modellen checken

> **Categorie:** development
> **Datum:** 2026-03-18
> **Sessie-context:** Upgrade van Gemini 2.5 Flash naar 3.1 Pro voor taxonomy classificatie
> **Relevantie:** midden
> **Reviewed:** 2026-03-18

## Inzicht

Bij hardnekkige LLM-kwaliteitsproblemen kan een model-upgrade effectiever zijn dan eindeloze prompt tuning. Gemini 3.1 Pro loste classificatieproblemen op die 8 iteraties prompt tuning met 2.5 Flash niet konden oplossen.

## Context

De taxonomy classifier haalde met Gemini 2.5 Flash na 8 prompt-iteraties BB ~70% en GR ~65-69% precision — nooit beide tegelijk boven 70%. Overstap naar Gemini 3.1 Pro (februari 2026) gaf direct BB 77% en GR 74% — beide boven de drempel, stabiel over meerdere runs.

## Geleerd

### Wat werkte
- Checken welke modellen beschikbaar zijn (de default was 10 maanden oud)
- Model ID via environment variable (`GEMINI_CLASSIFY_MODEL`) zodat wisselen makkelijk is
- Combinatie: betere definities + nieuwer model gaf het beste resultaat

### Wat niet werkte
- Eindeloos prompt tunen op een ouder model (diminishing returns na ~5 iteraties)
- Aannemen dat het geconfigureerde model nog steeds het beste is

### Waarom
Nieuwere LLM-generaties verbeteren met name op instruction following en nuanced reasoning — precies de vaardigheden die nodig zijn voor taxonomie-classificatie met complexe "Niet:"-grenzen. De 3.1-serie is "reasoning-first", wat classificatie met subtiele distincties verbetert.

## Toepassing

1. Check bij kwaliteitsproblemen altijd eerst of er nieuwere modellen beschikbaar zijn
2. Maak het model configureerbaar via environment variable
3. Na ~3-5 prompt-iteraties zonder verbetering: probeer een model-upgrade
4. Houd een referentie bij van beschikbare model IDs en hun release-data
