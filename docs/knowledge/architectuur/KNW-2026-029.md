# KNW-2026-029: CSS custom property cascade als componenttheming

> **Categorie:** architectuur
> **Datum:** 2026-04-01
> **Sessie-context:** Per-BB accentkleuren doorvoeren in 8 detail-componenten zonder props
> **Relevantie:** hoog
> **Reviewed:** 2026-04-01

## Inzicht

Een enkele CSS custom property (`--accent`) op een wrapper-element laat alle geneste componenten automatisch hun kleur erven via de CSS cascade. Dit vervangt prop-drilling, context APIs of styled-components — puur met native CSS.

## Context

Elke Building Block detail-pagina moest in zijn eigen accentkleur verschijnen: hero, checklist-vinkjes, evidence-blok, prose-kopjes, CTA-gradient en quickStart-nummers. In totaal 8 componenten die dezelfde kleur moesten gebruiken.

## Geleerd

### Wat werkte
- `<div class="bb-page" style="--accent: var(--bb-prompt-design);">` op de pagina-wrapper
- Componenten gebruiken `var(--accent, var(--teal))` — de fallback garandeert backwards compatibility
- Geen enkele component hoeft een kleur-prop te ontvangen
- Werkt met Astro's scoped styles (de custom property doorbreekt de scope boundary)

### Wat niet werkte
- Initieel probeerden we de map in zowel JS (voor de wrapper) als CSS (voor de homepage grid) te definiëren — dubbele bron. De /simplify review ving dit op.

### Waarom
CSS custom properties worden geërfd via de DOM-boom, net als `color` of `font-size`. Ze respecteren geen component-scope grenzen. Dit maakt ze ideaal voor theming: je zet de waarde hoog in de boom en alles eronder past zich aan.

## Toepassing

Gebruik dit patroon wanneer meerdere componenten op een pagina dezelfde variabele waarde moeten delen die per pagina-variant verschilt. Voorbeelden: kleurthema per entity, spacing-varianten per layout, of typografie-aanpassingen per sectie. Definieer de mapping in een shared lib-bestand en gebruik die zowel voor de wrapper (JS) als voor standalone grids (CSS data-attributen).
