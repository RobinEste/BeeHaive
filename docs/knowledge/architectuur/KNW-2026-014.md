# KNW-2026-014: asyncio.run() bridge patroon voor sync→async in gemengde contexten

> **Categorie:** architectuur
> **Datum:** 2026-03-17
> **Sessie-context:** Ingestion pipeline met dual ingest: sync CLI-pipeline die async LightRAG ainsert() moet aanroepen
> **Relevantie:** hoog

## Inzicht

`asyncio.run()` crasht met `RuntimeError` als er al een event loop draait (bijv. in FastAPI/uvicorn). Gebruik een `_run_async()` helper die eerst checkt of er een loop actief is: zo nee, gebruik `asyncio.run()`; zo ja, draai een nieuwe loop in een `ThreadPoolExecutor`.

## Context

De ingestion pipeline is sync (CLI-first), maar moet async `ainsert()` aanroepen voor LightRAG. Oorspronkelijk werd `asyncio.run()` direct aangeroepen. Dit werkte perfect in CLI-modus, maar zou crashen als de pipeline later vanuit een FastAPI endpoint wordt aangeroepen (waar uvicorn al een event loop draait).

## Geleerd

### Wat werkte
- `_run_async()` helper met `asyncio.get_running_loop()` check
- `ThreadPoolExecutor` als fallback voor bestaande event loop
- Het patroon is generiek en herbruikbaar voor elke sync→async bridge

### Wat niet werkte
- Direct `asyncio.run()` aanroepen — crasht in FastAPI context
- De crash werd pas ontdekt door de /review agents, niet bij handmatig testen (CLI werkte prima)

### Waarom
Python's `asyncio.run()` maakt altijd een **nieuwe** event loop aan. Als er al eentje draait (zoals in uvicorn), faalt het. De ThreadPoolExecutor-aanpak draait `asyncio.run()` in een aparte thread, die zijn eigen event loop heeft.

## Toepassing

Gebruik dit patroon wanneer:
- Een sync functie async code moet aanroepen
- De sync functie zowel vanuit CLI als vanuit een async webframework kan worden aangeroepen
- De async call niet op de hot path zit (ThreadPoolExecutor heeft overhead)

```python
def _run_async(coro):
    try:
        asyncio.get_running_loop()
    except RuntimeError:
        return asyncio.run(coro)
    else:
        import concurrent.futures
        with concurrent.futures.ThreadPoolExecutor(max_workers=1) as pool:
            return pool.submit(asyncio.run, coro).result()
```
