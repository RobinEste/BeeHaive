# KNW-2026-031: data-attribuut patroon voor entity-specifieke CSS-variabelen

> **Categorie:** architectuur
> **Datum:** 2026-04-01
> **Sessie-context:** Kleur-per-BB en kleur-per-GR implementeren op homepage grid en timeline
> **Relevantie:** hoog
> **Reviewed:** 2026-04-01

## Inzicht

Het patroon `data-entity="slug"` + CSS `[data-entity="slug"] { --accent: var(--token); }` koppelt entity-specifieke styling aan HTML-elementen zonder inline styles of JavaScript. In combinatie met een shared lib die dezelfde slugs exporteert, voorkom je dubbele bronnen.

## Context

Zowel de Building Blocks grid (7 items) als de Guardrail timeline (7 items) moesten per-entity kleuren krijgen. De grid gebruikt `data-bb="knowledge"`, de timeline `data-gr="human-agency"`. Beide zetten een lokale CSS custom property (`--accent` resp. `--gr-accent`) die door alle child-selectors wordt gebruikt.

## Geleerd

### Wat werkte
- `data-bb={block.slug}` in de Astro template — clean, geen inline styles
- CSS attribute selectors zijn performant en specifiek genoeg
- De lokale `--accent` variabele houdt de rest van de CSS DRY — hover, glow, border, nummer-kleur gebruiken allemaal dezelfde variabele
- Gradient timeline-lijn met alle 7 GR-kleuren geeft een visueel rijker resultaat dan monochroom

### Wat niet werkte
- Initieel bestond de slug→kleur mapping zowel in de CSS (attribute selectors) als in een JS-object (`bbAccentMap` in `[slug].astro`). De /simplify review ving de duplicatie op → map geëxtraheerd naar `lib/bb-accents.ts`

### Waarom
CSS heeft geen loop-construct, dus de 7 attribute selectors zijn onvermijdelijk. Maar de JS-map kan wel gedeeld worden, zodat nieuwe entities slechts op twee plekken hoeven: de shared lib en de CSS-selectors in het component.

## Toepassing

Gebruik dit patroon voor elk lijstcomponent waar items een unieke visuele identiteit nodig hebben: categorieën, statussen, prioriteiten, tags. Definieer de kleur-tokens in `globals.css` (`:root`), de slug→token mapping in een shared lib, en de CSS-koppeling via `[data-*]` selectors in het component.
