# KNW-2026-011: Frontier API boven lokaal LLM voor classificatietaken

> **Categorie:** architectuur
> **Datum:** 2026-03-09
> **Sessie-context:** Pivot van vLLM+Qwen naar Gemini API voor ingestion pipeline taxonomy mapping
> **Relevantie:** hoog

## Inzicht

Voor gestructureerde classificatietaken (tekst → voorgedefinieerde categorieën) is een frontier LLM via API bijna altijd beter dan een lokaal gehost model. De infra-overhead van lokaal (GPU management, quantization, model updates, OOM crashes) weegt niet op tegen de voordelen, tenzij je offline moet draaien of extreme latency-eisen hebt.

## Context

De ingestion pipeline moest tekst classificeren naar 7 BuildingBlocks, 7 Guardrails, Topics en Authors. Het oorspronkelijke plan gebruikte vLLM-MLX met Qwen 3.5 9B (4-bit quantized) lokaal. Na meerdere sessies bleek dit instabiel: OOM errors, quantization-artefacten, model versie mismatches. De classificatietaak is relatief simpel (structured output van ~20 categorieën) en vereist geen lage latency.

## Geleerd

### Wat werkte
- Frontier LLM (Gemini 2.5 Flash): betrouwbaar, snelle structured output, geen infra-overhead
- PII-redactie vóór de API call lost het privacy-bezwaar op
- Sync SDK (`google-genai`) — geen async complexiteit nodig

### Wat niet werkte
- Lokaal LLM voor een taak die slechts af en toe draait (CLI-first, niet real-time)
- De extra LightRAG entity extraction tussenstap die nodig was omdat het lokale model niet krachtig genoeg was voor directe classificatie

### Waarom
Lokaal LLM loont bij: hoog volume (kosten), offline vereiste, extreme latency-eisen, of wanneer ruwe data niet naar een externe API mag. Voor een CLI-tool die af en toe draait met al-geredacteerde tekst is geen van die voorwaarden van toepassing.

## Toepassing

Beslisboom voor lokaal vs. API LLM:
- **API als:** laag volume, geen offline-eis, data is geredacteerd/publiek, structured output voldoet
- **Lokaal als:** hoog volume (>1000 calls/dag), offline must-have, onge-redacteerde PII, of real-time latency (<100ms)
- **Start met API**, migreer naar lokaal als volumes of latency dat vereisen — niet andersom
